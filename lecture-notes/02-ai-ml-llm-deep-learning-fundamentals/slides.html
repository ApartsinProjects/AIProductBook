Module 2

AI, ML, LLM, and Deep Learning Fundamentals

The minimum model intuition an AI engineer needs for the rest of the course.

  • A category map
  • Token and embedding intuition
  • Transformer intuition
  • Metric and threshold judgment

Orientation

How the concepts fit together

Main intuition

These are nested ideas, not interchangeable buzzwords.

Why it matters

Without this map, later modules feel disconnected.

See also: lecture notes: large-picture map.
This is the conceptual backbone of the module.2
Landscape

AI, ML, deep learning, transformers, and LLMs are nested concepts

AI

Broad umbrella for intelligent behavior.

ML

Learning from data.

Deep learning

Neural representation learning.

Transformers / LLMs

Attention-based architectures and large pretrained language models.

Key correction

“LLM” is not a synonym for “AI.”

See also: lecture notes: landscape.
Category clarity prevents a lot of confusion later.3
Learning

How the model gets its learning signal matters

Supervised

Input + desired label.

Unsupervised

Find structure in raw data.

Self-supervised

Data creates its own labels.

Reinforcement

Learn from reward and feedback.

Most important practical point

LLMs get broad competence mainly from self-supervised pretraining, then become more useful through post-training.

See also: lecture notes: learning paradigms.
Students should know what kind of signal created the behavior they see.4
Neural nets

Neural networks matter because they learn representations

Large-picture intuition

Layers transform raw input into a space where useful predictions become easier.

Why engineers care

This is why hidden states, embeddings, and semantic similarity become operationally useful.

Connection to Module 1

Task framing still matters, but the model is learning internal patterns rather than just memorizing outputs.

See also: lecture notes: neural networks.
Representation learning is the conceptual bridge to embeddings.5
Transformers

Transformers changed modern AI because context changes meaning

Attention intuition

Meaning depends on what other tokens matter right now.

Engineering implication

Prompt shape and context assembly directly affect behavior.

See also: lecture notes: transformers.
This is the root of prompt and context engineering.6
Representations

Tokens are the model’s units. Embeddings are its semantic representations.

Tokens

Not the same as words. They affect cost, latency, and context fit.

Embeddings

Vector representations where related meanings tend to end up closer together.

Why this matters

Token counts shape prompt cost. Embeddings shape semantic search.

See also: lecture notes: tokens and embeddings and lab exercise 1.
This slide tees up RAG and retrieval later in the course.7
Lifecycle

Broad competence and user-facing helpfulness come from different stages

Pretraining

Build broad language competence.

Post-training

Shape helpfulness and alignment.

Inference

Use the model inside a real product.

Key correction

RLHF does not simply “make the model smart.” It mainly shapes user-facing behavior.

See also: lecture notes: training lifecycle.
Students should distinguish competence from alignment.8
Metrics

Precision, recall, thresholds, and calibration are really cost decisions

Precision

When we act, how often are we right?

Recall

How many real cases do we catch?

Calibration

Do confidence scores mean what they claim?

Deep insight

The right threshold depends on the cost of false positives versus false negatives.

See also: lecture notes: metrics and thresholds and lab exercise 3.
One threshold does not fit every product.9
Limits

Powerful models are still probabilistic and brittle

Hallucination

Plausible but ungrounded output.

Brittleness

Small context shifts can change behavior a lot.

Uncertainty

Confidence is often imperfect.

Operational realism

Respect the model’s capability, but never mistake it for guaranteed truth.

See also: lecture notes: failures and Module 1 failure modes.
This is why retrieval, evals, and guardrails exist later.10
Concrete

What the fundamentals look like in engineering work

Token intuition

same visible length != same token cost

Threshold intuition

lower threshold -> more recall
higher threshold -> more precision

Practical lesson

Fundamentals become useful when they change system choices: chunking, retrieval, routing, review, and cost controls.

See also: lecture notes: examples.
This is the bridge from theory to production design.11
Wrap-up

What students should now be able to do

Name the category

AI vs ML vs DL vs transformer vs LLM.

Name the mechanism

Token, embedding, attention, post-training, threshold.

Name the consequence

How it affects cost, quality, retrieval, or product risk.
