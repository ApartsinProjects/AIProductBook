20.2 Autonomous Products: From Apps to Agents

The fundamental paradigm of software is shifting. For decades, we've built "apps" - tools that users operate to accomplish tasks. The next paradigm is "Agents" - AI systems that operate on users' behalf to accomplish tasks. This section explores what this shift means and how to prepare for it.

Why This Matters

Apps require users to know what to do. Agents figure out what to do. The difference is profound: apps are tools that extend human capability, Agents are delegates that replace human effort. Products that remain "apps" in an "agent" world risk becoming obsolete.

Analogy: Travel Agent vs. Booking Website

Think of the difference between a travel agent (an agent) and a booking website (an app). With a travel agent, you say "I want a relaxing week in June under $2,000" and they figure out flights, hotels, activities - they do the work. With a booking website, you must search for flights, compare prices, book the flight, search for hotels, compare, book, research restaurants, make reservations - you do all the work. The agent figure out what to do; the app waits for you to tell it what to do.

Fun Note: The Agent That Booked a Flight to Nowhere

An early agent system was given a budget and told to book a flight. It found a "flight" for $50 that turned out to be a train route mistyped as a flight. The agent had no way to verify真实性 (authenticity). No human would have been fooled. Agents need better bullshit detectors than humans do.

Imagine If...

Imagine asking your current email app "Schedule my week" and it not only understanding what meetings you have but also declining unnecessary ones, proposing times for open items, updating your calendar, and sending the necessary confirmations - all without you clicking anything. That's an agent. Your current email app is an app. The difference is whether you specify actions or outcomes.

Real-World Scenario: The Productivity Shift

Consider planning a trip. Current apps: search for flights on Google Flights, compare prices, book on airline site, search for hotels on Booking.com, book, research restaurants on Yelp, make reservations, etc.

Agent future: "Plan me a week in Barcelona in June under $3,000, include flights, hotel, and restaurant recommendations." AI does everything: finds flights, books, researches neighborhood, makes reservations, builds itinerary. One prompt, complete result.

Three Types of Autonomous Products

Type 1: Autonomous Workflow Automation

The AI takes over multi-step processes that currently require human orchestration.

Example: Autonomous Scheduling

Current: User finds meeting times, sends invites, manages back-and-forth for rescheduling, sends reminders.

Agent: "Schedule a 1-hour meeting with the marketing team next week." AI finds times that work, sends invites, handles responses, manages conflicts, sends calendar invites and reminders.

Type 2: Research Synthesis Agents

The AI digests large amounts of information and produces actionable insights.

Example: Market Research Agent

Current: Analyst spends days reading reports, articles, competitor websites, then synthesizes findings into presentation.

Agent: "What's the state of the electric vehicle market? Who are the key players? What are the trends for the next 5 years?" AI analyzes hundreds of sources, identifies themes, creates comprehensive report with source citations.

Type 3: End-to-End Task Completion

The AI handles the complete lifecycle of a task, from initiation to completion.

Example: Customer Service Agent

Current: User contacts support, describes issue, agent investigates, resolves or escalates. Multiple touchpoints.

Agent: "Resolve this customer's billing issue." AI has access to account data, can issue refunds, update records, send confirmations. Problem solved in one interaction, no human needed unless the AI determines it's beyond its capabilities.

Building for the Agent Future

Design Principles for Agent Products

Skill Demonstration: Agent Architecture Pattern
// Core agent architecture
class Agent {
  constructor() {
    this.planner = new TaskPlanner();      // Breaks down goals
    this.executor = new ToolExecutor();   // Uses available tools
    this.evaluator = new ResultEvaluator(); // Checks outcomes
    this.memory = new ContextMemory();    // Maintains state
  }
  
  async achieve(goal, context) {
    // 1. Understand what success looks like
    const successCriteria = await this.planner.define(goal);
    
    // 2. Create execution plan
    const plan = await this.planner.create(successCriteria, context);
    
    // 3. Execute step by step
    let result;
    for (const step of plan.steps) {
      result = await this.executor.run(step, this.memory);
      
      // 4. Evaluate after each step
      if (!this.evaluator.isOnTrack(result, successCriteria)) {
        // 5. Re-plan if needed
        plan = await this.planner.replan(result, context);
      }
    }
    
    // 6. Final verification
    return this.evaluator.finalize(result, successCriteria);
  }
}

Key Technical Capabilities

  • Tool use: Agents need to interact with external systems - APIs, databases, other software
  • Long-running state: Agents handle multi-step tasks that span minutes or days
  • Context management: Maintaining relevant context across long interactions
  • Error recovery: Autonomous recovery when things go wrong
  • Transparency: Users need to understand what the agent is doing
  • Control: Users need to constrain or override agent behavior
Real-World Scenario: Building an Agent Today

A company building a "research analyst" agent started with: a web search tool, a document reading tool, a note-taking tool, and an LLM. Their agent architecture:

  • User provides research question
  • LLM breaks it into sub-questions
  • Agent searches for each sub-question in parallel
  • Agent reads top results
  • LLM synthesizes findings
  • Agent formats into report with citations
  • User can ask follow-up questions

The key insight: they didn't need new technology. They needed to compose existing tools with agentic orchestration.

The Transition Timeline

PhaseTimelineProducts
Current 2024-2025 AI-assisted apps (LLM in the interface)
Transition 2025-2026 Semi-autonomous (AI handles subtasks)
Emerging 2026-2027 Agentic products (AI handles complete workflows)
Mature 2027+ Fully autonomous (complex goal completion)
Don't Skip the Transition

You can't go from app to agent overnight. The path is: assisted app (LLM helps users) -> augmented automation (AI automates sub-tasks) -> agent (AI handles complete workflows). Each phase builds the capabilities for the next.

Key Takeaways
  • Apps require users to specify what to do; Agents figure it out
  • Three types: workflow automation, research synthesis, task completion
  • Build tool-use capability early
  • Agent architecture requires planning, execution, evaluation, memory
  • Transition through phases: assisted -> augmented -> agentic -> autonomous
  • Start with composing existing tools, not new technology
