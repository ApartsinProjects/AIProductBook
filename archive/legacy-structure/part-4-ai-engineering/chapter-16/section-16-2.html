16.2 Guardrails and Fallbacks

Objective: Production harden your AI systems for scale and reliability.

Prerequisites: Completion of Section 16.1 recommended.
Big Picture: Safety Enables Trust

This section covers guardrails that ensure AI systems operate within safe boundaries and fallbacks that enable graceful degradation. In the context of AI engineering, these patterns are essential for production systems. Without guardrails, AI may generate harmful content; without fallbacks, a single failure cascades to all users.

Fun Note: The Guardrail That Stopped Shakespeare

A content moderation AI had a guardrail blocking "harmful language." It blocked Shakespeare, Mark Twain, and the US Constitution. The word "murder" in Hamlet triggered the filter. Guardrails need exceptions for literature, education, and news. Context matters as much as content.

16.2 Guardrails and Fallbacks

Guardrails ensure AI systems operate within safe boundaries. Fallbacks ensure graceful degradation when things go wrong. Together, they make your system resilient to unexpected inputs and failures.

Why This Matters

AI models are unpredictable. Without guardrails, they may generate harmful content, reveal sensitive information, or take unintended actions. Without fallbacks, a single failure cascades to all users. Both are essential for production systems.

Watch Out: Guardrails Can Be Bypassed

A common misconception is that guardrails are bulletproof. Sophisticated attackers can craft inputs designed to bypass your filters (adversarial attacks). Input and output guardrails reduce risk but do not eliminate it. For high-stakes applications, you need human-in-the-loop for sensitive operations and ongoing monitoring for new bypass techniques.

HealthMetrics: PII Detection and Redaction

HealthMetrics must prevent PII from entering prompts due to HIPAA requirements:

import re
from typing import List

class PHIDetector:
    """Detect Personal Health Information in text"""
    
    def __init__(self):
        self.patterns = {
            'ssn': r'\b\d{3}-\d{2}-\d{4}\b',
            'mrn': r'\bMRN[:\s]+\d{6,}\b',
            'dob': r'\b(DOB|Date of Birth)[:\s]+(\d{1,2}[/-]\d{1,2}[/-]\d{2,4}|\w+ \d{1,2},? \d{4})\b',
            'phone': r'\b\d{3}[-.]?\d{3}[-.]?\d{4}\b',
            'email': r'\b[\w.-]+@[\w.-]+\.\w+\b',
            'name': r'\b(Dr\.|Mr\.|Mrs\.|Ms\.)\s+[A-Z][a-z]+\s+[A-Z][a-z]+\b',
        }
    
    def detect(self, text: str) -> List[PHIEntity]:
        found = []
        for phi_type, pattern in self.patterns.items():
            for match in re.finditer(pattern, text):
                found.append(PHIEntity(
                    type=phi_type,
                    text=match.group(),
                    start=match.start(),
                    end=match.end()
                ))
        return found
    
    def redact(self, text: str, entities: List[PHIEntity]) -> str:
        result = text
        for entity in sorted(entities, key=lambda e: e.start, reverse=True):
            result = result[:entity.start] + f"[{entity.type.upper()}]" + result[entity.end:]
        return result

# Usage in HealthMetrics query pipeline
async def sanitize_query(user_input: str) -> str:
    detector = PHIDetector()
    entities = detector.detect(user_input)
    if entities:
        logger.warning(f"PHI detected in query: {[e.type for e in entities]}")
        return detector.redact(user_input, entities)
    return user_input

HealthMetrics found that 3% of queries contained accidental PHI (like "my patient John Smith's MRN"). The redaction system catches these cases before they enter the prompt, preventing potential HIPAA violations.

1. Input Guardrails

Input guardrails filter harmful prompts before they reach the model. This prevents prompt injection attacks, limits abuse, and reduces costs from malicious use.

Practical Example: Input Filtering
class InputGuardrails:
    """Filter harmful inputs before processing"""
    
    def __init__(self):
        self.blocked_patterns = [
            "ignore previous instructions",
            "system prompt",
            "you are now",
        ]
        self.safety_classifier = load_safety_model()
        self.max_length = 10000
    
    async def check(self, user_input: str) -> GuardrailResult:
        # Check length
        if len(user_input) > self.max_length:
            return GuardrailResult(
                allowed=False,
                reason="Input exceeds maximum length"
            )
        
        # Check for prompt injection patterns
        for pattern in self.blocked_patterns:
            if pattern.lower() in user_input.lower():
                return GuardrailResult(
                    allowed=False,
                    reason="Contains blocked pattern"
                )
        
        # Run safety classifier
        safety = await self.safety_classifier.classify(user_input)
        if not safety.safe:
            return GuardrailResult(
                allowed=False,
                reason=f"Safety violation: {safety.category}"
            )
        
        return GuardrailResult(allowed=True)

2. Output Guardrails

Output guardrails filter model responses before returning them to users. This catches harmful content, sensitive information, and ensures quality standards.

Skill Demonstration: Output Filtering
class OutputGuardrails:
    """Filter model outputs before returning"""
    
    def __init__(self):
        self.pii_detector = load_pii_detector()
        self.content_filter = load_content_filter()
    
    async def check(self, output: str, context: dict) -> GuardrailResult:
        # Detect PII
        pii_found = await self.pii_detector.detect(output)
        if pii_found:
            # Redact or block based on policy
            output = self.redact_pii(output, pii_found)
        
        # Check content policy
        content_check = await self.content_filter.classify(output)
        if not content_check.allowed:
            return GuardrailResult(
                allowed=False,
                reason=f"Content policy violation: {content_check.category}",
                replacement=self.get_safe_alternative(context)
            )
        
        # Check for hallucinated facts (if context provided)
        if context.get("verify_facts"):
            verified = await self.verify_facts(output, context["sources"])
            if not verified.all_valid:
                output = self.add_disclaimer(output, verified.invalid_facts)
        
        return GuardrailResult(allowed=True, output=output)
    
    def redact_pii(self, text: str, pii_locations: list) -> str:
        # Replace PII with [REDACTED]
        for pii in pii_locations:
            text = text.replace(pii.text, "[REDACTED]")
        return text

3. Fallback Strategies

Fallbacks ensure the system degrades gracefully when errors occur. Users should always get some response, even if it's not ideal.

Case Study: Multi-Layer Fallback
async def generate_with_fallback(prompt: str) -> str:
    """Multi-layer fallback strategy"""
    
    try:
        # Try primary model
        return await primary_llm.generate(prompt)
    
    except RateLimitError:
        # Layer 1: Try fallback model
        try:
            return await fallback_llm.generate(prompt)
        except Exception:
            pass
    
    except TimeoutError:
        # Layer 2: Return cached response
        cached = await cache.get(prompt)
        if cached:
            return cached
    
    except ModelError as e:
        # Layer 3: Try simpler prompt
        simpler_prompt = simplify(prompt)
        try:
            return await primary_llm.generate(simpler_prompt)
        except Exception:
            pass
    
    # Final fallback: Return polite error with next steps
    return "I apologize, but I'm experiencing high demand. 
    Please try again in a few moments, or contact support for immediate assistance."

4. Human-in-the-Loop (HITL)

For high-stakes decisions, route to humans for approval. This ensures critical actions have human oversight.

Real-World Scenario: Approval Workflow
async def process_request(request: UserRequest) -> Response:
    """Human-in-the-loop for sensitive actions"""
    
    # Determine if human approval needed
    if needs_human_approval(request):
        # Create approval ticket
        ticket = await create_approval_ticket(
            user=request.user,
            action=request.action,
            details=request.details,
            ai_recommendation=request.recommendation
        )
        
        return Response(
            requires_approval=True,
            ticket_id=ticket.id,
            message="This action requires manager approval. 
            A ticket has been created and your manager will be notified."
        )
    
    # Low-risk: proceed automatically
    return await execute_action(request)

def needs_human_approval(request: UserRequest) -> bool:
    """Determine if approval is needed"""
    high_risk_actions = [
        "refund_over_1000",
        "delete_data",
        "modify_permissions",
        "access_sensitive"
    ]
    
    return (
        request.action in high_risk_actions or
        request.amount > APPROVAL_THRESHOLD or
        request.user.risk_level == "high"
    )
