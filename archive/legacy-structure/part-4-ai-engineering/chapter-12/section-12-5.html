12.5 Advanced Agent Patterns

Basic Agents that simply call tools in sequence are the starting point, not the destination. Production AI systems require more sophisticated patterns that enable self-correction, planning, and continuous improvement. This section covers the agent patterns that separate prototype AI from production AI.

Why This Matters

A basic agent can execute a fixed plan. An advanced agent can evaluate its own performance, correct its course, plan multi-step solutions, and learn from experience. The difference is reliability. Advanced patterns are what make Agents trustworthy enough for production.

The Reliability Gap

A basic agent might succeed 60% of the time in demos. Advanced agent patterns can push that to 95% or higher. In production, a 60% success rate is a support ticket tsunami. The patterns in this section close that gap.

1. ReAct Pattern: Reasoning Plus Acting

ReAct (Reasoning + Acting) combines explicit reasoning with tool actions. The agent generates reasoning traces that inform its actions, and actions that provide new information for reasoning. This creates a feedback loop between thought and action.

Case Study: ReAct for Complex Research

Without ReAct, an agent might make a plan and execute it blindly. With ReAct, it thinks about what it knows, decides what to try, observes the result, and adjusts:

# ReAct agent loop
while not done:
    # Think: What do I know? What should I try next?
    thought = reasoner.generate_thought(
        context=conversation_history,
        question=original_goal,
        observations=previous_results
    )
    
    # Act: Take an action based on reasoning
    if thought.action == "search":
        result = search_tool.invoke(thought.query)
    elif thought.action == "browse":
        result = browse_tool.invoke(thought.url)
    elif thought.action == "calculate":
        result = calculate_tool.invoke(thought.expression)
    
    # Observe: Update what we know
    observations.append({"thought": thought, "result": result, "action": thought.action})
    
    # Check if we can answer now
    if enough_information(observations):
        answer = reasoner.generate_answer(observations)
        done = True
Code Example: ReAct Implementation
from langchain.Agents import Agent
from langchain.prompts import PromptTemplate
from langchain.schema import AgentAction, AgentFinish

class ReActAgent:
    """ReAct pattern: Alternate between reasoning and acting"""
    
    def __init__(self, tools: list, llm):
        self.tools = {t.name: t for t in tools}
        self.llm = llm
        self.prompt = PromptTemplate.from_template("""
You are a research assistant using the ReAct pattern.

Question: {input}

Follow this format:
Thought: What should I do next? What information do I need?
Action: search/browse/calculate/lookup/[tool_name]
Action Input: The input to the tool
Observation: The result of the action
... (repeat until you can answer)
Thought: I can now answer the question.
Answer: [your answer here]

Begin:
""")
    
    def run(self, question: str) -> str:
        observations = []
        max_iterations = 10
        
        for _ in range(max_iterations):
            # Generate thought and action
            response = self.llm.invoke(
                self.prompt.format(input=question) +
                "\n".join([f"Observation: {o}" for o in observations])
            )
            
            # Parse response
            thought, action, action_input = self._parse_response(response)
            
            if thought.lower().startswith("i can now"):
                return action  # Final answer
            
            # Execute action
            if action in self.tools:
                result = self.tools[action].invoke(action_input)
                observations.append(f"{action}: {result}")
            else:
                observations.append(f"Unknown action: {action}")
        
        return "Could not complete within iteration limit"

2. Reflection Agent: Self-Evaluation Pattern

Reflection Agents evaluate their own outputs before returning them. The agent generates a response, then critiques it, then improves it. This creates a quality check that catches errors before they reach users.

Why Reflection Matters

A basic agent generates once. A reflection agent generates, evaluates, and refines. The critique catches hallucinations, logical errors, and formatting issues that the initial generation missed. For high-stakes outputs, this extra step is essential.

Practical Example: Reflection for Code Generation
class ReflectionAgent:
    """Agent that critiques and improves its own outputs"""
    
    def __init__(self, code_llm, critique_llm, tools):
        self.code_llm = code_llm
        self.critique_llm = critique_llm
        self.tools = tools
    
    def generate_and_reflect(self, task: str) -> str:
        # Step 1: Initial generation
        initial_code = self.code_llm.invoke(f"""
            Write Python code for: {task}
            Follow best practices and include error handling.
        """)
        
        # Step 2: Self-critique
        critique = self.critique_llm.invoke(f"""
            Review this code for:
            1. Correctness - will it produce the right output?
            2. Security - any vulnerabilities?
            3. Performance - any efficiency issues?
            4. Error handling - are edge cases covered?
            
            Code to review:
            {initial_code}
            
            Provide specific issues found, or "Looks good" if no issues.
        """)
        
        # Step 3: Decide if revision needed
        if "looks good" in critique.lower():
            return initial_code
        
        # Step 4: Revise based on critique
        revised_code = self.code_llm.invoke(f"""
            Original task: {task}
            
            Original code:
            {initial_code}
            
            Critique:
            {critique}
            
            Write the improved code addressing the critique.
        """)
        
        return revised_code

3. Planning Agents: Decompose and Execute

Planning Agents break complex tasks into subgoals and execute them in sequence. The agent maintains a plan state, tracks completed steps, and adapts the plan as it learns what works.

Real-World Scenario: Planning for Multi-Step Research

A research task like "Analyze the competitive landscape for electric vehicles" requires planning:

class PlanningAgent:
    """Agent that maintains and executes a plan"""
    
    def __init__(self, llm, tools):
        self.llm = llm
        self.tools = tools
        self.plan = []
        self.completed = []
    
    def create_plan(self, goal: str) -> list[str]:
        """Break goal into executable steps"""
        response = self.llm.invoke(f"""
            Break down this goal into ordered steps:
            Goal: {goal}
            
            Format:
            1. [First step]
            2. [Second step]
            ...
            
            Steps should be ordered, with each step depending on previous ones.
        """)
        
        # Parse steps from response
        self.plan = self._parse_steps(response)
        return self.plan
    
    def execute_plan(self) -> dict:
        """Execute plan, adapting as needed"""
        results = {}
        
        for i, step in enumerate(self.plan):
            print(f"Executing step {i+1}: {step}")
            
            # Execute step with context from previous results
            context = "\n".join([f"Step {j+1}: {results[j]}" for j in range(i)])
            result = self._execute_step(step, context)
            
            # Evaluate if step achieved its goal
            evaluation = self._evaluate_step(step, result)
            
            if not evaluation.success:
                # Replan from this point
                new_steps = self._replan(step, evaluation.reason)
                self.plan = self.plan[:i+1] + new_steps + self.plan[i+1:]
            
            results[i] = result
            self.completed.append(step)
        
        return self._synthesize_results(results)

4. Supervisor Pattern: Hierarchical Control

The supervisor pattern uses a high-level agent to coordinate multiple specialized sub-Agents. The supervisor does not do the work; it delegates to specialists and manages the flow between them.

Case Study: Supervisor for Customer Service
class CustomerServiceSupervisor:
    """Supervisor that routes to specialized Agents"""
    
    def __init__(self):
        self.classifier = load_intent_classifier()
        self.billing_agent = BillingAgent()
        self.technical_agent = TechnicalAgent()
        self.escalation_agent = EscalationAgent()
        self.general_agent = GeneralAgent()
    
    def handle(self, customer_message: str, customer_data: dict) -> str:
        # Step 1: Classify intent
        intent = self.classifier.classify(customer_message)
        
        # Step 2: Route to appropriate specialist
        if intent.category == "billing":
            response = self.billing_agent.handle(
                message=customer_message,
                account=customer_data["account"]
            )
        elif intent.category == "technical":
            response = self.technical_agent.handle(
                message=customer_message,
                product=customer_data["product"]
            )
        elif intent.needs_escalation:
            response = self.escalation_agent.handle(
                message=customer_message,
                reason=intent.escalation_reason
            )
        else:
            response = self.general_agent.handle(customer_message)
        
        # Step 3: Log for monitoring
        self.log_interaction(intent, response)
        
        return response

5. Error Recovery Patterns

Production Agents must handle failures gracefully. Error recovery patterns give Agents strategies for when things go wrong.

Skill Demonstration: Error Recovery with Retry Strategies
class ErrorRecoveryAgent:
    """Agent with systematic error recovery"""
    
    def __init__(self, base_agent):
        self.agent = base_agent
        self.max_retries = 3
    
    def execute_with_recovery(self, task: str) -> str:
        last_error = None
        
        for attempt in range(self.max_retries):
            try:
                # Try the main approach
                return self.agent.execute(task)
            
            except ToolNotFoundError as e:
                # Strategy: Simplify tool usage
                task = self._simplify_task(task, e.tool_name)
                last_error = e
            
            except RateLimitError as e:
                # Strategy: Wait and retry with backoff
                wait_time = 2 ** attempt  # Exponential backoff
                sleep(wait_time)
                last_error = e
            
            except TimeoutError as e:
                # Strategy: Reduce scope
                task = self._reduce_scope(task)
                last_error = e
            
            except InvalidResponseError as e:
                # Strategy: Rephrase the request
                task = self._rephrase_request(task)
                last_error = e
        
        # All retries exhausted - use fallback
        return self._fallback_response(task, last_error)

Combining Patterns

The most capable production Agents combine multiple patterns. A research agent might use ReAct for decision-making, reflection for output quality, planning for complex tasks, and error recovery for robustness. Choose the combination that matches your reliability requirements.

# Learner adaptation logic
def adapt_learning_path(learner_id: str, progress: dict) -> LearningPath:
    learner = get_learner(learner_id)
    
    if progress.completion_rate < 0.6:
        # Learner struggling - add prerequisites
        return LearningPath(
            additional_topics=[prerequisite_skills],
            pace="slower",
            practice_ratio=0.7  # More practice problems
        )
    elif progress.completion_rate > 0.9 and progress.quiz_scores > 0.85:
        # Learner excelling - accelerate
        return LearningPath(
            skip_review=True,
            pace="faster",
            advanced_topics=True
        )
    else:
        # On track - standard path
        return LearningPath(pace="standard")
