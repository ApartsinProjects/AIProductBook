12.6 Agent Memory Management

Objective: Learn how to manage memory and state in long-running conversational Agents.

Prerequisites: Completion of Section 12.5 recommended. Understanding of agent architectures.
Big Picture: Memory Enables Continuity

This section covers memory management strategies for long-running conversational Agents, from sliding windows to knowledge graphs. In the context of AI products, memory enables Agents to maintain context across interactions, which is essential for useful customer-facing applications. Without proper memory management, Agents either forget important context or degrade over time.

12.6 Agent Memory Management

AI Agents that serve users over multiple interactions must remember relevant history while discarding noise. Memory management is the discipline that makes this possible. Without proper memory management, Agents either forget important context or accumulate unbounded state that degrades performance.

Why This Matters

A customer service agent that does not remember your name from the last interaction provides a frustrating experience. A research agent that remembers every token from every conversation eventually runs out of Context Window. Memory management is the difference between Agents that are useful over time and Agents that fail in extended interactions.

The Unbounded Memory Problem

Passing entire conversation histories to language models creates three problems: escalating latency as history grows, escalating cost as more tokens are processed, and degrading quality as relevant context is diluted by noise. Memory management is not optional; it is essential for viable long-running Agents.

1. Conversation Window Memory

The simplest approach: maintain a sliding window of recent messages. When the window fills, the oldest messages are discarded. This is effective for most conversational applications.

Practical Example: Sliding Window Implementation
class SlidingWindowMemory:
    """Maintain last N messages in conversation"""
    
    def __init__(self, max_tokens: int = 16000):
        self.messages = []
        self.max_tokens = max_tokens  # Leave room for response
    
    def add_message(self, role: str, content: str):
        self.messages.append({"role": role, "content": content})
        self._prune_if_needed()
    
    def _prune_if_needed(self):
        while self.count_tokens(self.messages) > self.max_tokens:
            # Remove oldest message
            self.messages.pop(0)
    
    def get_context(self) -> list:
        return self.messages
    
    def count_tokens(self, messages: list) -> int:
        # Approximate token count
        return sum(len(m["content"].split()) * 1.3 for m in messages)

2. Summarization-Based Memory

For longer conversations, periodically summarize the history into a compact form. This preserves key information while dramatically reducing token count.

Skill Demonstration: Summarization Memory
class SummarizationMemory:
    """Periodically summarize conversation history"""
    
    def __init__(self, llm, max_messages: int = 20, summary_interval: int = 10):
        self.messages = []
        self.summary = ""
        self.llm = llm
        self.max_messages = max_messages
        self.summary_interval = summary_interval
        self.messages_since_summary = 0
    
    def add_message(self, role: str, content: str):
        self.messages.append({"role": role, "content": content})
        self.messages_since_summary += 1
        
        if self.messages_since_summary >= self.summary_interval:
            self._summarize()
            self.messages_since_summary = 0
    
    def _summarize(self):
        """Compress old messages into summary"""
        old_messages = self.messages[:-self.summary_interval]
        if not old_messages:
            return
        
        # Generate summary of old messages
        summary_prompt = f"""
        Summarize this conversation concisely, preserving key facts and decisions:
        
        {self._format_messages(old_messages)}
        
        Summary:
        """
        
        new_summary = self.llm.invoke(summary_prompt)
        
        # Keep summary and recent messages
        self.summary = f"{self.summary}\n{new_summary}" if self.summary else new_summary
        self.messages = self.messages[-self.summary_interval:]
    
    def get_context(self) -> list:
        context = []
        if self.summary:
            context.append({"role": "system", "content": f"Previous conversation summary: {self.summary}"})
        context.extend(self.messages)
        return context

3. Entity Memory

Track key entities (people, companies, preferences, facts) that emerge during conversation. Entity memory enables Agents to recall specific details without storing entire conversations.

Case Study: Entity Memory for Customer Support
class EntityMemory:
    """Remember key entities across conversations"""
    
    def __init__(self, max_entities: int = 100):
        self.entities = {}  # name -> {value, last_updated, importance, category}
        self.max_entities = max_entities
    
    def add_entity(self, name: str, value: str, category: str = "general", importance: float = 0.5):
        self.entities[name] = {
            "value": value,
            "last_updated": datetime.now(),
            "importance": importance,
            "category": category
        }
        self._evict_if_needed()
    
    def get_entity(self, name: str) -> str | None:
        entity = self.entities.get(name)
        if entity:
            entity["last_updated"] = datetime.now()  # Bump on access
            return entity["value"]
        return None
    
    def _evict_if_needed(self):
        if len(self.entities) > self.max_entities:
            # Remove lowest importance entities
            sorted_entities = sorted(
                self.entities.items(),
                key=lambda x: (x[1]["importance"], x[1]["last_updated"])
            )
            
            # Remove bottom 10%
            to_remove = len(sorted_entities) // 10
            for name, _ in sorted_entities[:to_remove]:
                del self.entities[name]
    
    def extract_and_store(self, text: str):
        """Extract entities from text using LLM"""
        entities = self.llm.invoke(f"""
            Extract key entities from this text as JSON:
            {{
              "entities": [
                {{"name": "entity name", "value": "what it refers to", "category": "person/product/fact/preference", "importance": 0.1-1.0}}
              ]
            }}
            
            Text: {text}
        """)
        # Store extracted entities
        for entity in entities:
            self.add_entity(entity["name"], entity["value"], entity.get("category"), entity.get("importance", 0.5))
    
    def get_context_string(self) -> str:
        if not self.entities:
            return ""
        
        # Group by category
        by_category = {}
        for name, data in self.entities.items():
            cat = data.get("category", "general")
            if cat not in by_category:
                by_category[cat] = []
            by_category[cat].append(f"{name}: {data['value']}")
        
        return "\n".join([f"{cat}: {', '.join(items)}" for cat, items in by_category.items()])

4. Knowledge Graph Memory

For complex domains, maintain a knowledge graph that captures relationships between entities. This enables reasoning over connected facts rather than isolated mentions.

Code Example: Knowledge Graph Memory
class KnowledgeGraphMemory:
    """Graph-based memory for related entities"""
    
    def __init__(self, graph_db):
        self.graph = graph_db  # Neo4j or similar
    
    def add_fact(self, subject: str, predicate: str, object: str, confidence: float = 1.0):
        """Add a fact to the graph"""
        self.graph.query("""
            MERGE (s:Entity {name: $subject})
            MERGE (o:Entity {name: $object})
            MERGE (s)-[r:RELATES {type: $predicate, confidence: $confidence}]->(o)
        """, {"subject": subject, "predicate": predicate, "object": object, "confidence": confidence})
    
    def query(self, subject: str = None, predicate: str = None, related_to: str = None) -> list:
        """Query the graph for facts"""
        if subject and predicate and related_to:
            results = self.graph.query("""
                MATCH (s:Entity {name: $subject})-[r:RELATES {type: $predicate}]->(o:Entity {name: $related_to})
                RETURN s.name, r.type, o.name, r.confidence
            """, {"subject": subject, "predicate": predicate, "related_to": related_to})
        elif subject:
            results = self.graph.query("""
                MATCH (s:Entity {name: $subject})-[r]->(o)
                RETURN o.name, r.type, r.confidence
            """, {"subject": subject})
        
        return results
    
    def get_context_for_agent(self, topic: str) -> str:
        """Get all facts related to a topic for agent context"""
        facts = self.query(related_to=topic)
        return "\n".join([f"- {f['s.name']} {f['r.type']} {f['o.name']}" for f in facts])

5. Hybrid Memory Architecture

Production systems typically combine multiple memory types. Each serves a different purpose and operates at a different granularity.

Real-World Scenario: Hybrid Memory for Research Assistant
class HybridAgentMemory:
    """
    Production memory system combining multiple memory types.
    Each operates at different granularity for different purposes.
    """
    
    def __init__(self, llm):
        # Working memory: recent conversation (sliding window)
        self.working = SlidingWindowMemory(max_tokens=8000)
        
        # Episodic memory: summarized past interactions
        self.episodic = SummarizationMemory(llm, max_messages=20)
        
        # Semantic memory: key facts and entities
        self.semantic = EntityMemory(max_entities=100)
        
        # Procedural memory: learned patterns and workflows
        self.procedural = {}
    
    def add_turn(self, user_message: str, agent_response: str):
        """Add a conversation turn to appropriate memories"""
        # Working memory
        self.working.add_message("user", user_message)
        self.working.add_message("assistant", agent_response)
        
        # Episodic memory
        self.episodic.add_message("user", user_message)
        self.episodic.add_message("assistant", agent_response)
        
        # Semantic memory: extract entities
        self.semantic.extract_and_store(user_message)
        self.semantic.extract_and_store(agent_response)
    
    def get_agent_context(self) -> str:
        """Combine all memory types into context string"""
        parts = []
        
        # Recent conversation
        working_context = self.working.get_context()
        if working_context:
            parts.append("Recent conversation:\n" + self._format_messages(working_context))
        
        # Past summary
        episodic_context = self.episodic.get_context()
        if episodic_context:
            parts.append("Past interactions:\n" + episodic_context)
        
        # Key entities
        entity_context = self.semantic.get_context_string()
        if entity_context:
            parts.append("Known facts:\n" + entity_context)
        
        return "\n\n".join(parts)
    
    def remember_learned_pattern(self, pattern_name: str, pattern_data: dict):
        """Store learned procedural knowledge"""
        self.procedural[pattern_name] = pattern_data
    
    def get_learned_pattern(self, pattern_name: str) -> dict | None:
        return self.procedural.get(pattern_name)

Memory Is Not Free

Every memory operation has a cost. Summarization uses LLM calls. Entity extraction uses LLM calls. Knowledge graph updates add latency. Design your memory system with the same cost discipline you apply to the rest of your architecture. More memory is not always better.
