Chapter 12

Prompts, Chains, Agents, and Orchestration

Building Blocks of AI Systems

You need your AI to do more than answer a single prompt. You need it to reason through a multi-step problem, fetch data from external APIs, maintain context across a conversation, and coordinate multiple components. One prompt will not get you there.

Complex AI systems are not built from single prompts. They are built from prompts composed into chains, chains that become agents, and agents that orchestrate together. This chapter is your guide to that architecture.

Objective: Master the building blocks of AI systems: prompts, chains, agents, and orchestration frameworks.

Chapter Overview

This chapter covers the fundamental building blocks of AI systems. You will learn advanced prompting patterns like chain-of-thought and few-shot learning, the distinction between chains and agents, the MCP and A2A protocols for agent communication, and how agents use tools to interact with external systems. These skills are essential for building any AI application beyond simple single-prompt interactions.

The key insight is that prompt engineering is just the beginning. To build sophisticated AI applications, you need to understand how to compose multiple prompts, handle different types of user requests, and orchestrate multiple AI components working together. This chapter provides that foundation.

Learning Objectives

  • Apply advanced prompting patterns (CoT, few-shot, role, constraint)
  • Distinguish between chains and agents for different use cases
  • Implement MCP and A2A protocols for agent communication
  • Build agents that can use external tools and APIs
  • Design orchestration frameworks for complex AI workflows

Prerequisites

Readers should have completed Chapter 11 on core principles. Familiarity with Python and basic API usage is expected. Experience with any LLM API (OpenAI, Anthropic, etc.) is helpful. If you are new to prompting, focus on Section 12.1 first.
