Big Picture: Tools Extend Agent Capabilities

12.4 Agent Skills and Tool Use

Modern Agents can use tools to interact with external systems, extending their capabilities beyond pure text generation. Tools transform Agents from passive responders into active participants that can take actions in the world. Understanding how to define, implement, and manage tools is essential for building capable AI systems.

Why This Matters

Without tools, an agent is just a sophisticated text generator. With tools, an agent can retrieve real-time information, execute code, interact with APIs, and take actions that affect the real world. The combination of agent reasoning and tool execution is what makes AI systems genuinely useful for production applications.

Core Tool Categories

1. Web Search and Browsing

Search tools allow Agents to access current information beyond their training data. This is essential for questions about recent events, rapidly changing domains, or information that may have changed since the model's knowledge cutoff.

Practical Example: Search Tool Implementation
from langchain.Agents import Tool
from langchain_community.utilities import SerpAPIWrapper

search = SerpAPIWrapper()

search_tool = Tool(
    name="web_search",
    func=search.run,
    description="""Use this tool for questions about current events, 
    recent developments, or real-time information. 
    Input should be a search query."""
)

# Agent can now use this to answer "What happened in tech this week?"
# The agent decides when to use the tool based on the question

2. Code Execution

Code execution tools enable Agents to run code, perform calculations, and even build and test software. This is crucial for Agents that need to analyze data, run simulations, or help with software development tasks.

Skill Demonstration: Python REPL Tool
from langchain_experimental.tools import PythonREPLTool

python_tool = PythonREPLTool()

# The agent can execute Python code for:
# - Data analysis and visualization
# - Mathematical calculations
# - Text processing
# - Testing code snippets

# Example usage in an agent task:
# "Calculate the moving average of these stock prices and plot the result"
# Agent: 1) Uses code tool to load data
#        2) Computes moving average
#        3) Creates visualization
#        4) Describes findings to user

3. API Calls

API tools connect Agents to external services, databases, and enterprise systems. This enables integration with your existing infrastructure, allowing Agents to look up customer data, create tickets, send notifications, or interact with business systems.

Case Study: CRM Integration
import requests
from langchain.tools import StructuredTool

def get_customer(customer_id: str) -> dict:
    """Get customer information from CRM"""
    response = requests.get(
        f"https://api.crm.example.com/customers/{customer_id}",
        headers={"Authorization": f"Bearer {API_KEY}"}
    )
    return response.json()

def create_ticket(customer_id: str, title: str, description: str) -> dict:
    """Create a support ticket"""
    response = requests.post(
        "https://api.crm.example.com/tickets",
        json={
            "customer_id": customer_id,
            "title": title,
            "description": description,
            "priority": "medium"
        }
    )
    return response.json()

# Define tools for the agent
tools = [
    StructuredTool.from_function(get_customer),
    StructuredTool.from_function(create_ticket),
]

# Agent can now handle support requests end-to-end:
# 1. Look up customer
# 2. Understand the issue
# 3. Create a ticket with the right details

4. File System Operations

File tools allow Agents to read, write, and manage files. This is essential for development tasks, document processing, and any application that needs to work with persistent data.

Code Example: File Operations
from langchain_community.agent_toolkits import FileManagementToolkit

toolkit = FileManagementToolkit(
    root_dir="/project",
    allowed_directories=["/project/docs", "/project/data"]
)

# Available tools:
# - read_file
# - write_file  
# - list_directory
# - file_search
# - move_file
# - delete_file

# Agent can now:
# - Read code files to understand a codebase
# - Write generated documentation
# - Search through file contents
# - Organize project files

Tool Definition Best Practices

The Art of Tool Descriptions

The description field is critical. It tells the agent when to use the tool and how. A good description explains: (1) what the tool does, (2) when to use it, (3) what input it expects. Vague descriptions lead to Agents not using tools or using them incorrectly.
