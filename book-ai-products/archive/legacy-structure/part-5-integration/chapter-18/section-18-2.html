18.2 Architecture Comparison: Managed vs. Self-Hosted

When building a Perplexity-like search tool, one of your first decisions is whether to use managed services (APIs from OpenAI, Pinecone, etc.) or self-host open-source alternatives. This section compares both approaches with real trade-offs.

Why This Matters

This choice affects your costs, control, scaling ceiling, and team composition. A startup choosing managed services can move fast but face high costs at scale. Self-hosting reduces costs but requires more engineering investment.

Analogy: Renting vs. Buying a House

Think of managed services like renting an apartment: you pay monthly, you can move in immediately, you call the landlord when something breaks, but you never build equity and rent can increase. Self-hosting is like buying a house: upfront costs are higher, you're responsible for maintenance, but over time costs are lower and you have full control. Most startups "rent" first (managed), then "buy" when they have the resources (self-host).

💡 Aha Moment: The Migration Pattern

The shock: Most successful AI companies start with managed services and migrate to self-hosting when they hit scale. The pattern is nearly universal.

What this means: Do not premature-optimize for self-hosting. Start with APIs, validate product-market fit, and only migrate when costs force the issue. The engineering cost of self-hosting rarely pays off until you are processing tens of millions of tokens per month.

Fun Note: The Migration That Saved Nothing

A company proudly migrated from managed APIs to self-hosted models, expecting massive savings. After 3 months of engineering time, GPU costs, and DevOps overhead, they had saved $2,000/month but were spending $15,000/month in engineering. They migrated back to managed services. Sometimes the math just does not work out.

Detailed Component Analysis

1. Search Index

Managed: Algolia

Pros: Fast (<50ms), excellent typo tolerance, easy to use

Cons: Expensive at scale, no semantic search, vendor lock-in

Cost: $0.03-0.15 per 1K searches

Self-Hosted: SearXNG + Qdrant

Pros: Free, full control, combines keyword + vector

Cons: More ops overhead, requires maintenance

Cost: Infrastructure only (~$200/month for small deployment)

Real-World Scenario: The Reranking Decision

A team building a medical search tool compared reranking approaches. The managed Cohere Rerank API cost ~$0.10 per 1K queries. Running a local cross-encoder model cost ~$0.01 per 1K but required GPU. For 1M queries/month, managed cost $100 vs. self-hosted cost $10 + $200 GPU = $210. At lower volumes, managed wins; at higher volumes, self-hosted makes sense.

Decision Framework: Which Approach to Choose?

Choose Managed If:
  • Launching in 4-8 weeks is critical
  • Team has 1-3 engineers (not ML specialists)
  • Traffic is < 100K queries/month
  • Flexibility to switch models is important
Choose Self-Hosted If:
  • Cost optimization is critical
  • Have ML/DevOps expertise
  • Data privacy requires full control
  • Traffic exceeds 500K queries/month
Real-World Scenario: Hybrid Approach

A fintech company used a hybrid approach: managed for embeddings and generation (fast iteration), self-hosted for vector storage (data privacy requirements). They kept the most sensitive data in their own infrastructure while leveraging managed services for the AI components. This gave them the best of both worlds.

Migration Path: Start Managed, Migrate Later

Most successful teams start with managed services and migrate to self-hosted as they scale. Here's a practical migration path: 

// Migration path with abstractions
class SearchBackend {
  // Use strategy pattern to swap implementations
  constructor(strategy: 'managed' | 'self-hosted') {
    if (strategy === 'managed') {
      this.retriever = new AlgoliaRetriever();
      this.llm = new OpenAIClient();
    } else {
      this.retriever = new SearXNGRetriever();
      this.llm = new OllamaClient();
    }
  }
}

// This allows gradual migration
// Month 1-3: Fully managed (fast launch)
// Month 4-6: Migrate storage to self-hosted
// Month 7-9: Migrate reranking to self-hosted
// Month 10+: Migrate generation (if needed)
Key Takeaways
  • Managed = faster to launch, easier to use, more expensive at scale
  • Self-hosted = more control, lower cost at scale, more ops overhead
  • Most teams should start with managed and migrate as they scale
  • Use abstraction layers to make migration easier later
  • Consider hybrid approaches based on specific constraints
