5.2 Automated Testing and CI/CD

Testing is where vibe coding delivers some of its biggest productivity gains, and also where it requires the most careful oversight. AI can generate unit tests, integration tests, and end-to-end tests at speeds that would be impossible manually. However, the tests AI generates are only as good as the specifications it is given. You must clearly define what behavior you expect, what constitutes a pass versus a failure, and what edge cases matter for your specific use case.

5.2.1 Test-Driven Vibe Coding

The most effective approach is test-driven vibe coding: write test descriptions first, then let AI generate the implementation and the tests in parallel. This ensures that your understanding of correct behavior is captured before implementation begins, providing a clear benchmark for evaluating the AI-generated code. When tests fail, you immediately know whether the failure is due to incorrect implementation or incorrect test expectations.

Skill Demonstration: Writing Test Descriptions for AI

The key to getting good tests from AI is providing precise specifications. Here's a template that works well:

// Test specification template for AI
const testSpec = `
Generate unit tests for a function that processes user checkout.
The function is in file: checkout.js

Function signature:
processCheckout(cart: Cart, paymentMethod: PaymentMethod, shippingAddress: Address): Promise<CheckoutResult>

Expected behavior:
1. Returns success=true when payment is processed successfully
2. Returns success=false with error="INSUFFICIENT_STOCK" when any item is out of stock
3. Returns success=false with error="INVALID_ADDRESS" when address validation fails
4. Applies discount codes: "SAVE10" = 10% off, "SAVE20" = 20% off (only one per order)
5. Returns order ID on success
6. Throws error if cart is empty

Test cases needed:
- Happy path: valid cart, valid payment, valid address
- Empty cart: should throw "Cart cannot be empty"
- Out of stock item: should return INSUFFICIENT_STOCK
- Invalid address (missing postal code): should return INVALID_ADDRESS
- Expired payment method: should return error code
- Valid discount code "SAVE10": should apply 10% discount
- Valid discount code "SAVE20": should apply 20% discount
- Multiple discount codes: should use the higher discount
- Invalid discount code: should be ignored (no discount applied)
- Network error on payment: should return "PAYMENT_FAILED"

Use Jest. Include descriptive test names following the pattern: should [expected behavior] when [condition]
`;

Real-World Scenario: E-Commerce Checkout Testing

A team building an e-commerce platform used test-driven vibe coding for their checkout flow. They wrote comprehensive test specifications covering 47 different scenarios, then used AI to generate both the implementation and the tests simultaneously.

When they ran the tests, 12 failed. Instead of manually debugging, they asked AI to analyze the failures. In 8 cases, the test expectations were wrong (they had made mistakes in their specifications). In 4 cases, the implementation had bugs. They fixed the specification, regenerated the implementation, and had a passing test suite in 2 hours.

Without this approach, they estimated it would have taken 2 days of manual test writing and debugging to reach the same coverage.

5.2.2 AI-Generated Unit Tests

AI excels at generating unit tests because it understands code patterns and can identify boundary conditions that humans might miss. Given a function to test, AI can generate tests for normal cases, edge cases, error conditions, and boundary values.

Skill Demonstration: Generating Unit Tests from Code

// Prompt for unit test generation
const unitTestPrompt = `
Generate comprehensive unit tests for the following function using Jest:

function calculateShippingCost(weight: number, destination: string, speed: string): number {
    const baseRates = {
        'US': 5.99,
        'CA': 12.99,
        'UK': 14.99,
        'EU': 16.99,
        'OTHER': 24.99
    };
    
    const baseRate = baseRates[destination] || baseRates['OTHER'];
    const weightMultiplier = weight <= 1 ? 1 : 1 + (weight - 1) * 0.5;
    const speedMultiplier = speed === 'express' ? 2 : 1;
    
    return baseRate * weightMultiplier * speedMultiplier;
}

Requirements:
1. Test each destination with weight=1 and speed='standard'
2. Test weight boundaries: 0, 1, 2, 5, 10, 100
3. Test both 'standard' and 'express' speed
4. Test unknown destination (should use 'OTHER' rate)
5. Test invalid inputs (negative weight, empty destination)
6. Use describe blocks to group by destination region
7. Include edge case: weight=0.5
`;

5.2.4 CI/CD Integration

CI/CD integration extends AI assistance into the deployment pipeline. AI can review pull requests for common issues like security vulnerabilities, missing tests, or style violations. Some teams have configured AI Agents that run on every push, performing security audits on code changes and flagging potential issues before they reach production.

Skill Demonstration: Setting Up AI Code Review in CI/CD

# GitHub Actions workflow for AI code review
name: AI Code Review
on: [pull_request]

jobs:
  ai-review:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
        with:
          ref: ${{ github.event.pull_request.head.sha }}
          fetch-depth: 0
      
      - name: AI Code Review
        env:
          OPENAI_API_KEY: ${{ secrets.OPENAI_API_KEY }}
        run: |
          # Get the diff
          DIFF=$(git diff --no-color origin/main...HEAD -- '*.ts' '*.js' '*.py')
          
          # Send to AI for review
          curl -X POST https://api.openai.com/v1/chat/completions \
            -H "Authorization: Bearer $OPENAI_API_KEY" \
            -H "Content-Type: application/json" \
            -d '{
              "model": "gpt-4",
              "messages": [{
                "role": "system",
                "content": "You are a senior software engineer reviewing code changes. Focus on: security vulnerabilities, performance issues, potential bugs, code quality, and test coverage. Provide specific, actionable feedback."
              }, {
                "role": "user",
                "content": "Review the following code changes:\n\n" + env.DIFF
              }],
              "temperature": 0.3
            }'

Real-World Scenario: Automated PR Review at Scale

A mid-size team with 12 developers implemented AI code review in their CI pipeline. They configured three levels of feedback:

  • Critical (blocks merge): Security vulnerabilities, potential data loss, authentication bypass
  • Warning (non-blocking): Code style violations, missing comments, possible performance issues
  • Suggestion (informational): Refactoring opportunities, alternative approaches, improvements

In the first month, AI caught 47 issues that would have reached the main branch, including 3 potential SQL injection vulnerabilities and 12 cases of missing error handling. Developer satisfaction with code review increased because routine issues were caught automatically, allowing human reviews to focus on architectural decisions.

5.2.5 AI-Powered Security Scanning

Security scanning represents a particularly valuable application of AI in the CI/CD pipeline. AI-powered static analysis goes beyond traditional pattern matching to understand code context and identify subtle security issues like injection vulnerabilities, authentication bypasses, and data exposure risks. These tools are not perfect, but they catch a significant percentage of common vulnerabilities and free security experts to focus on more complex audits.

Skill Demonstration: AI Security Review for Authentication Code

// Prompt for security review of authentication code

Review the following authentication code for security vulnerabilities:

async function login(email, password, req) {
    const user = await db.query('SELECT * FROM users WHERE email = ?', email);

Skill Demonstration: Comprehensive Security Scanner Script

#!/bin/bash
# ai-security-scan.sh - Run AI-powered security scan on code changes

set -e

echo "Running AI-powered security scan..."

# Get changed files
CHANGED_FILES=$(git diff --name-only origin/main...HEAD)
SECURE_FILES=$(echo "$CHANGED_FILES" | grep -E '\.(js|ts|py|java|go|rb)$')

# Scan each file
for file in $SECURE_FILES; do
    echo "Scanning $file..."
    
    CONTENT=$(cat "$file")
    
    # Send to AI for security analysis
    SECURITY_REPORT=$(curl -s -X POST https://api.openai.com/v1/chat/completions \
        -H "Authorization: Bearer $OPENAI_API_KEY" \
        -H "Content-Type: application/json" \
        -d "{
            \"model\": \"gpt-4\",
            \"messages\": [{
                \"role\": \"system\",
                \"content\": \"You are a security expert. Analyze code for OWASP Top 10 vulnerabilities. Provide findings in JSON format.\"
            }, {
                \"role\": \"user\",
                \"content\": \"Analyze this code for security vulnerabilities in file $file:\n\n$CONTENT\"
            }],
            \"temperature\": 0.2
        }")
    
    # Parse and report findings
    echo "$SECURITY_REPORT" | jq -r '.choices[0].message.content'
    
    # Check for critical issues
    if echo "$SECURITY_REPORT" | jq -r '.choices[0].message.content' | grep -q "Critical"; then
        echo "::error::Critical security issues found in $file"
        exit 1
    fi
done

echo "Security scan complete."
