Big Picture: MCP as AI Connectivity Standard

MCP standardizes how AI assistants connect to external services, creating an ecosystem where connectors are reusable across different AI tools. This breaks down the barrier between local development environments and production infrastructure, enabling AI to work across your entire stack. MCP is to AI what APIs were to web services: a transformative connectivity standard.

Fun Note: USB-C for AI

USB-C unified charging cables in 2016 after decades of proprietary solutions. MCP is attempting the same for AI integrations, though the number of proprietary AI connector formats suggests we are still in the "mini-USB era."

6.4 MCP Connectors and Integrations

While skills extend Claude Code's capabilities through prompts, MCP (Model Context Protocol) connectors extend its reach to external services. MCP is an open protocol that lets AI assistants connect to databases, APIs, and tools that live outside your local environment. Think of MCP as USB-C for AI, providing a standardized way to connect any service to any AI tool.

The power of MCP lies in its standardization. Before MCP, each AI assistant had to build custom integrations for every service. A database integration for Claude Code was completely different from one for ChatGPT. MCP changes this: you build the integration once, and any AI tool that supports MCP can use it. This creates an ecosystem where connectors are reusable and composable.

What is MCP?

The Model Context Protocol (MCP) is a standardized interface for connecting AI assistants to external resources. Rather than each AI tool implementing custom integrations for every service, MCP provides a common protocol. When you connect an MCP server, Claude Code gains new capabilities without needing to understand the specifics of each service.

MCP servers are separate processes that expose resources and tools via the MCP protocol. Claude Code connects to these servers and can then read from resources (like files or database queries) and call tools (like creating issues or sending messages). This architecture keeps your AI environment secure while enabling powerful integrations.

How MCP Works

# The MCP Architecture

┌─────────────────┐       MCP Protocol       ┌─────────────────┐
│                 │ ◄───────────────────────► │                 │
│  Claude Code    │                          │   MCP Server    │
│                 │   Resources (read)       │                 │
│  - Local files  │ ◄───────────────────────► │  - Database    │
│  - Your code    │                          │  - APIs         │
│  - Shell        │   Tools (execute)         │  - Services     │
│                 │ ◄───────────────────────► │                 │
└─────────────────┘                          └─────────────────┘

# What Claude Code sees:
- New "resources" it can read (database tables, API responses)
- New "tools" it can call (create_issue, query_database)
- All standardized through MCP

Why This Matters

Without MCP, connecting Claude Code to your production database would require custom code. With MCP, you install a server, configure it once, and your AI assistant can query databases, create tickets, send notifications, and more. The difference is like the difference between writing a custom API client versus using a standard library.

Key Insight

MCP enables a new workflow: AI that acts across your entire stack. Your AI assistant can now query production databases, create Jira tickets, deploy to Kubernetes, and send Slack notifications, all within a single conversation. This breaks down the wall between your local development environment and the rest of your infrastructure.

Popular MCP Connectors

The MCP ecosystem has grown rapidly, with connectors for almost every service developers use. Here are the most essential connectors that will transform your workflow.

mcp-k8s-go (378 stars)

What it does: Connects Claude Code to Kubernetes. You can query cluster status, inspect pods, view logs, and even apply configurations directly from your AI assistant.

Installation: 

# Install the Kubernetes MCP server
go install github.com/anomalyco/mcp-k8s-go@latest

# Add to Claude Code config (~/.claude.json):
{
  "mcpServers": {
    "kubernetes": {
      "command": "mcp-k8s-go",
      "env": {
        "KUBECONFIG": "~/.kube/config"
      }
    }
  }
}

Use case: Developers managing Kubernetes deployments. Debug production issues, check pod status, or deploy new versions without leaving your AI workflow.

Case Study: Debugging Production via AI

# Instead of SSH'ing into servers and manually checking:

User: "Check if there are any errors in the payment-service pods"
MCP Server: Returns pod list with status and recent events

User: "Show me the logs from the last 5 minutes"
MCP Server: Streams recent log output

User: "The payments are failing. What's the error?"
MCP Server: Shows the error message from the logs

User: "Scale the payment-service to 3 replicas and check if it helps"
MCP Server: Executes kubectl scale, confirms new replicas running

User: "Now check if payments are succeeding"
MCP Server: Queries logs for successful payment processing

This workflow that used to require multiple terminal sessions, kubectl configurations, and constant context switching now happens in a single conversation.

Configuring MCP Connectors

MCP connectors are configured in Claude Code's settings. The configuration specifies the server command and any required environment variables. Each MCP server has its own setup process, but the general pattern involves installing the server package and adding configuration to your Claude Code config file.

Configuration File Location

# Claude Code config location
# macOS/Linux: ~/.claude.json
# Windows: %USERPROFILE%\.claude.json

# Example configuration
{
  "mcpServers": {
    "kubernetes": {
      "command": "mcp-k8s-go",
      "env": {
        "KUBECONFIG": "~/.kube/config"
      }
    },
    "postgres": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-postgres"],
      "env": {
        "DATABASE_URL": "postgresql://user:pass@localhost:5432/mydb"
      }
    },
    "github": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-github"],
      "env": {
        "GITHUB_TOKEN": "ghp_xxxxx"
      }
    }
  }
}

Installation Patterns

# Pattern 1: Go-based servers
go install github.com/user/mcp-server@latest

# Pattern 2: npm-based servers
npm install -g @org/mcp-server
# or
npx -y @org/mcp-server

# Pattern 3: Python-based servers
pip install mcp-server-package
mcp-server-package --help

# Pattern 4: Docker-based servers
docker run -it --rm -e API_KEY=xxx org/mcp-server:latest
Security Consideration

MCP connectors can access external services on your behalf. Only connect to services you trust, and review what data the connector can access. Many connectors require read/write access to significant resources.

Security Best Practices

# 1. Use environment variables for secrets, never hardcode
{
  "mcpServers": {
    "github": {
      "command": "...",
      "env": {
        // Good: Reference env var
        "GITHUB_TOKEN": "env:GITHUB_TOKEN"
      }
    }
  }
}

# 2. Limit connector access to what's needed
// Instead of full admin access, create dedicated tokens
// with minimal permissions for each connector

# 3. Review what resources the connector can access
// Some connectors can read everything - be intentional

# 4. Disconnect connectors when not in use
// Especially for production databases

Building Custom MCP Servers

If you need to connect to a service without an existing MCP server, you can build your own. MCP servers can be written in any language and expose resources and tools via the protocol. The MCP specification defines how to structure your server and what messages to exchange.

When to Build Custom Connectors

Build Custom MCP If Use Existing Connector If
Service has no existing MCP server MCP server already exists
Need custom logic or transformations Default functionality meets your needs
Internal/proprietary service Public service with community support
Highly specific workflow needs General-purpose functionality is sufficient

Python MCP Server Example

# Example: A simple custom MCP server in Python
# Using the MCP SDK

from mcp.server import Server
from mcp.server.stdio import stdio_server
from mcp.types import Tool, TextResource
import asyncio

app = Server("my-custom-server")

@app.list_tools()
async def list_tools() -> list[Tool]:
    return [
        Tool(
            name="query_database",
            description="Query the internal analytics database",
            inputSchema={
                "type": "object",
                "properties": {
                    "sql": {"type": "string", "description": "SQL query to execute"}
                },
                "required": ["sql"]
            }
        ),
        Tool(
            name="get_dashboard_url",
            description="Get the URL for a specific dashboard",
            inputSchema={
                "type": "object",
                "properties": {
                    "dashboard_id": {"type": "string"}
                },
                "required": ["dashboard_id"]
            }
        )
    ]

@app.call_tool()
async def call_tool(name: str, arguments: dict) -> str:
    if name == "query_database":
        # Custom logic to query your database
        result = await query_analytics(arguments["sql"])
        return format_result(result)
    elif name == "get_dashboard_url":
        return f"https://analytics.company.com/d/{arguments['dashboard_id']}"
    
async def main():
    async with stdio_server() as streams:
        await app.run(
            streams[0],
            streams[1],
            app.create_initialization_options()
        )

if __name__ == "__main__":
    asyncio.run(main())

Building a custom MCP server is beyond the scope of this chapter, but the pattern is straightforward: implement the MCP protocol to expose your service's capabilities as resources and tools that Claude Code can consume.
