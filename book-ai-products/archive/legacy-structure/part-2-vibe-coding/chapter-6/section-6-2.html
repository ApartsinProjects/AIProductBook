Development Workflow Skills

These skills improve your daily development workflow, helping with planning, communication, and code understanding. They are the foundation of a productive Claude Code setup.

planning-with-files (17,808 stars)

What it does: Implements Manus-style persistent markdown planning. Creates a structured planning workflow where you can track tasks, decisions, and progress in Markdown files that persist across sessions. This is the workflow pattern behind the $2B AI coding agent acquisition.

Installation: 

cd ~/.claude/skills
git clone https://github.com/anomalyco/planning-with-files.git

Use case: Any complex project that needs persistent planning. Create a plan.md file and the skill helps you break down tasks, track progress, and maintain context across long development sessions.

Why this matters: Without persistent planning, you lose context between Claude Code sessions. This skill solves that problem by maintaining a living document that grows with your project. It's particularly valuable for multi-day feature development where you need to remember what you did yesterday.

Case Study: Using planning-with-files for a New Feature

Imagine you're building a user authentication system. You start by creating a plan.md file and the skill automatically structures your planning: 

# Feature: User Authentication System

## Goals
- Implement JWT-based authentication
- Add OAuth support for Google and GitHub
- Create secure password reset flow

## Tasks
- [ ] Design database schema for users
- [ ] Implement JWT token generation
- [ ] Create login/logout endpoints
- [ ] Add OAuth provider integration
- [ ] Build password reset email flow
- [ ] Write comprehensive tests

## Decisions
- Use bcrypt for password hashing
- Token expiry: 7 days
- Use Redis for session storage

## Progress
[Skill updates this as you progress]

The skill tracks your progress, suggests next steps, and helps you maintain focus. When you return to the project tomorrow, Claude Code reads the plan and knows exactly where you left off.

Use case: Security-critical projects. Use before deploying to production to catch common vulnerabilities.

Practical Example: Using Trail of Bits Skills

# Before deploying a new feature:
User: "Run security audit on this PR"
Skill: "I'll perform a security review of this code...

## Findings:

### Critical: SQL Injection Risk (line 42)
```python
query = f"SELECT * FROM users WHERE id = {user_id}"
```
This directly interpolates user input into SQL. Use parameterized queries instead.

### High: Hardcoded API Key (line 15)
```python
API_KEY = "sk-1234567890abcdef"
```
Move to environment variables or secrets manager.

### Medium: Missing Rate Limiting
The /login endpoint doesn't have rate limiting, vulnerable to brute force.

## Recommendations:
1. Use parameterized queries (use sqlalchemy ORM)
2. Move secrets to .env files
3. Add rate limiting with flask-limiter

Shall I implement these fixes?"

How to Evaluate New Skills

Before installing any skill, evaluate it on these criteria. First, check the repository stars and recent activity. Skills with more stars and recent commits are more likely to be maintained. Second, read the SKILL.md file to understand what the skill actually does. Look for clear, well-documented instructions. Third, check for tool usage. Skills that execute commands or access files should have clear security considerations. Fourth, test in a safe environment. Try the skill on a small task before relying on it for important work.

Evaluation Checklist

Before installing a skill, ask:

□ Does it have recent commits? (within last 3 months)
□ Is the SKILL.md well documented?
□ Does it explain what commands it runs?
□ Are there security concerns mentioned?
□ Does it match my workflow needs?
□ Have I tested it on a small task first?

Red flags to watch for:
- No commits in over 6 months
- Vague or missing SKILL.md
- Executes commands without explanation
- No documentation or examples
Security Warning

Skills can execute arbitrary commands on your system. Only install skills from trusted sources, and review the SKILL.md file before activation. Malicious skills could access sensitive data or damage your system.

Safe Installation Practices

# 1. Create a dedicated skills folder
mkdir -p ~/safe-claude-skills

# 2. Clone skills there first
cd ~/safe-claude-skills
git clone https://github.com/username/skill-name.git

# 3. Review the SKILL.md
cat ~/safe-claude-skills/skill-name/SKILL.md

# 4. Check for concerning tool definitions
# Look for: shell, exec, write, http_request
grep -E "(shell|exec|write|http)" ~/safe-claude-skills/skill-name/SKILL.md

# 5. Test on a simple task first
# Don't use it for production work until you've verified it works

# 6. Move to your main skills folder only after review
cp -r ~/safe-claude-skills/skill-name ~/.claude/skills/
