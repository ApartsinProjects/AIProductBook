Chapter 6

Claude Code Skills, Plugins, and Connectors

Organizing Your AI Development Toolkit

Objective: Learn to organize, share, and scale your AI development workflows with skills and connectors.

Chapter Overview

This chapter explores how to extend Claude Code beyond its out-of-the-box capabilities using skills, plugins, and connectors. You will learn the skill system architecture, how to discover and install community skills, how to build your own custom skills, and how to organize skills for your team. We also cover MCP (Model Context Protocol) connectors that let Claude Code interface with external services like GitHub, Jira, databases, and more. By the end of this chapter, you will have a personalized toolkit that supercharges your AI-assisted development workflow.

The skill system is what distinguishes a basic AI assistant from a fully customized AI development environment. Just as professional developers customize their IDEs with plugins, AI power users customize their AI tools with skills. The difference is that skills are written in natural language instructions (prompts), making them accessible to anyone who can write clear instructions.

Learning Objectives

  • Understand the Claude Code skill architecture and how it works
  • Discover and evaluate community skills from GitHub and marketplaces
  • Install and configure skills for your development workflow
  • Build custom skills for repeated tasks and team workflows
  • Connect Claude Code to external services via MCP connectors
  • Organize and manage skills at scale for teams

Prerequisites

This chapter builds on Chapter 3 (Understanding Vibe Coding) and Chapter 4 (Vibe Coding Workflows). You should be comfortable with Claude Code basics and have used it for several development sessions. Familiarity with command line tools and version control will help when we cover connector configuration.
