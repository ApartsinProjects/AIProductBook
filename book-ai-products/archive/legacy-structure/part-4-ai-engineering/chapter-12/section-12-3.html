12.3 MCP and A2A Protocols

The AI ecosystem is moving toward standardization, and two protocols are emerging as foundational: MCP (Model Context Protocol) and A2A (Agent-to-Agent Protocol). Understanding these protocols is essential for building interoperable, future-proof AI systems.

Why This Matters

Before these protocols, each tool and agent had its own API. Integrating a new tool meant writing custom integration code. MCP and A2A provide standardized ways for Agents to communicate with tools and with each other. This dramatically reduces integration overhead and enables building composable AI systems.

Watch Out: Not All Tools Support MCP

MCP is gaining adoption but not every tool supports it yet. Before designing your architecture around MCP, verify that your critical tools have MCP implementations. For tools without MCP, you'll still need custom integrations. A common mistake is assuming "everything supports MCP" and then discovering your key tool doesn't have an MCP server.

MCP (Model Context Protocol): The USB-C for AI

MCP standardizes how Agents connect to tools and data. Just as USB-C unified device connections, MCP aims to unify how AI systems access resources. Rather than writing custom code for each tool, you use MCP to define resources, prompts, and tools in a standardized format.

Skill Demonstration: Implementing MCP
from mcp.server import Server
from mcp.types import Tool, Resource

app = Server("my-research-agent")

# Define available resources (data sources)
@app.list_resources()
async def list_resources():
    return [
        Resource(
            uri="file:///docs",
            name="Documentation",
            description="Product documentation"
        ),
        Resource(
            uri="db://users",
            name="User Database",
            description="User profiles and preferences"
        )
    ]

# Define tools the agent can call
@app.list_tools()
async def list_tools():
    return [
        Tool(
            name="search_docs",
            description="Search through documentation",
            inputSchema={
                "type": "object",
                "properties": {
                    "query": {"type": "string"}
                },
                "required": ["query"]
            }
        ),
        Tool(
            name="get_user_context",
            description="Get user preferences for personalization",
            inputSchema={
                "type": "object",
                "properties": {
                    "user_id": {"type": "string"}
                },
                "required": ["user_id"]
            }
        )
    ]

@app.call_tool()
async def call_tool(name: str, arguments: dict):
    if name == "search_docs":
        return search_documentation(arguments["query"])
    elif name == "get_user_context":
        return get_user_preferences(arguments["user_id"])
    raise ValueError(f"Unknown tool: {name}")

With MCP, any agent that understands the protocol can connect to your tools without custom integration code.

A2A (Agent-to-Agent Protocol): Communication Between Agents

A2A enables communication between Agents built by different teams. With over 50 launch partners including Google, Anthropic, and Microsoft, A2A is emerging as the standard for multi-agent systems. A2A defines how Agents discover each other, negotiate capabilities, and exchange information.

Real-World Scenario: Multi-Team Agent Collaboration

Imagine a large organization where different teams build specialized Agents:

  • Data team: Agent that can query databases and run analytics
  • Research team: Agent that searches academic papers andsummarizes findings
  • Communication team: Agent that drafts emails and creates presentations

Without A2A, integrating these requires custom code. With A2A, each agent publishes its capabilities, and they can discover and communicate with each other automatically.

# Using A2A to connect Agents from different teams
from a2a.client import A2AClient

# Connect to the data team's analytics agent
data_agent = A2AClient.connect("agent://analytics.internal/capabilities")
data_agent.discover(capabilities=["query_sql", "run_analytics"])

# Connect to the research team's agent
research_agent = A2AClient.connect("agent://research.internal/capabilities")
research_agent.discover(capabilities=["search_papers", "summarize"])

# Have them collaborate on a task
task = {
    "description": "Analyze sales data and find relevant research on market trends",
    "requires": ["query_sql", "search_papers"],
    "output": "Report with data analysis and research context"
}

result = await data_agent.execute_task(
    task=task,
    sub_tasks=[research_agent]  # Automatically coordinate with research agent
)

Multi-Agent Orchestration Frameworks

Multi-agent orchestration frameworks provide the infrastructure for coordinating multiple AI Agents working together on complex tasks. These frameworks handle the complexity of agent communication, state management, and workflow orchestration, allowing developers to focus on agent behavior rather than infrastructure.

1. LangGraph

LangGraph, built by the creators of LangChain, provides a graph-based approach to orchestrating Agents. Agents are represented as nodes in a graph, with edges defining how control flows between them. This approach is particularly powerful for workflows that involve cycles, conditional logic, and dynamic routing.

Code Example: LangGraph Workflow
from langgraph.graph import StateGraph
from typing import TypedDict

class AgentState(TypedDict):
    messages: list
    current_task: str
    results: dict

workflow = StateGraph(AgentState)

# Add nodes
workflow.add_node("researcher", research_agent)
workflow.add_node("analyst", analysis_agent)
workflow.add_node("reporter", reporting_agent)

# Define edges
workflow.set_entry_point("researcher")

# Conditional routing based on task complexity
workflow.add_edge("researcher", "analyst")
workflow.add_conditional_edges(
    "analyst",
    lambda state: "reporter" if state["results"]["complexity"] == "high" else "reporter",
    {"reporter": "reporter"}
)

# Compile and run
graph = workflow.compile()
result = graph.invoke({
    "messages": [{"role": "user", "content": "Analyze market trends for Q4"}],
    "current_task": "market_analysis",
    "results": {}
})

LangGraph supports both human-in-the-loop patterns where humans approve agent actions before they execute, and parallel execution where multiple Agents work simultaneously on different parts of a task.

2. CrewAI

CrewAI provides a role-based orchestration framework where Agents are assigned specific roles within a "crew." Each role defines the agent's goal, backstory, and available tools. The framework handles coordination between roles, ensuring that Agents contribute to a coherent overall result rather than working at cross-purposes.

Case Study: Research Crew
from crewai import Agent, Crew, Task, Process

# Define Agents with specific roles
researcher = Agent(
    role="Senior Market Researcher",
    goal="Find comprehensive data on market trends",
    backstory="You have 15 years of experience in market analysis...",
    tools=[search_tool, database_tool]
)

analyst = Agent(
    role="Data Analyst",
    goal="Extract actionable insights from research",
    backstory="You specialize in turning raw data into insights...",
    tools=[analysis_tool, visualization_tool]
)

writer = Agent(
    role="Technical Writer",
    goal="Create clear, compelling reports",
    backstory="You excel at explaining complex topics...",
    tools=[document_tool]
)

# Define tasks
research_task = Task(
    description="Research Q4 2024 market trends in SaaS industry",
    agent=researcher,
    expected_output="Comprehensive research summary with sources"
)

analysis_task = Task(
    description="Analyze research for key insights and opportunities",
    agent=analyst,
    expected_output="List of actionable insights with data support"
)

writing_task = Task(
    description="Write executive summary report",
    agent=writer,
    expected_output="Professional report suitable for C-suite"
)

# Create crew with sequential process
crew = Crew(
    Agents=[researcher, analyst, writer],
    tasks=[research_task, analysis_task, writing_task],
    process=Process.sequential
)

# Execute
result = crew.kickoff()

CrewAI is particularly well-suited for tasks that require diverse expertise, like research teams where different Agents handle different aspects of information gathering and synthesis.

3. AutoGen

AutoGen from Microsoft provides a flexible framework for building applications with multiple Agents that can communicate with each other. The framework supports various conversation patterns including pairwise chat, group chat, and nested chat where Agents can spawn sub-conversations.

Code Example: AutoGen Group Chat
from autogen import ConversableAgent, GroupChat, GroupChatManager

# Create Agents with different personalities and capabilities
researcher = ConversableAgent(
    name="Researcher",
    system_message="You are a thorough researcher...",
    llm_config={"model": "gpt-4o"}
)

critic = ConversableAgent(
    name="Critic",
    system_message="You are a skeptical peer reviewer...",
    llm_config={"model": "gpt-4o"}
)

writer = ConversableAgent(
    name="Writer",
    system_message="You are a skilled technical writer...",
    llm_config={"model": "gpt-4o"}
)

# Create group chat
group_chat = GroupChat(
    Agents=[researcher, critic, writer],
    messages=[],
    max_round=5
)

# Create manager to orchestrate
manager = GroupChatManager(
    groupchat=group_chat,
    llm_config={"model": "gpt-4o"}
)

# Execute task
result = researcher.initiate_chat(
    manager,
    message="Write a research report on AI in healthcare"
)

AutoGen's strength is its flexibility: Agents can be customized with different LLMs, tools, and behaviors. The framework also supports both static agent definitions and dynamic agent creation at runtime.
