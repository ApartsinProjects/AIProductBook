Big Picture: Choosing Your Architecture

This section covers the fundamental distinction between chains (deterministic workflows) and Agents (dynamic decision-makers). In the context of AI product development, choosing the right pattern affects cost, reliability, and debuggability. The 10x cost difference between chains and Agents makes this one of the most important architectural decisions.

12.2 Chains vs Agents

Understanding the distinction between chains and Agents is fundamental to designing AI systems. Chains are deterministic sequences where each step follows from the previous one in a fixed order. Agents are dynamic systems where the model decides which actions to take based on the context. This fundamental difference affects everything from development approach to production reliability.

Why This Matters

Choosing the wrong pattern leads to either over-engineering (using Agents for simple tasks) or under-engineering (using chains for tasks that need flexibility). The cost and complexity difference is significant: a chain is typically 10x cheaper to run than an equivalent agent, and much easier to debug. But using a chain where you need dynamic adaptation leads to brittle systems that fail in unexpected ways.

💡 Aha Moment: The 10x Cost Difference

The shock: A chain is typically 10x cheaper to run than an equivalent agent. The cost and complexity difference is massive.

What this means: Do not reach for an agent unless you genuinely need dynamic decision-making. If the workflow is predictable, use a chain. Many teams over-engineer by defaulting to Agents when a simple chain would suffice, and they pay 10x more than necessary.

Chains: Deterministic Workflows

Chains are ideal for simple transformations where the path from input to output is well-defined and predictable. Text summarization, translation, sentiment analysis, and similar tasks are well-suited to chain-based approaches because they have clear steps and expected outputs.

Practical Example: A Translation Chain
from langchain import LLMChain
from langchain.prompts import PromptTemplate
from langchain.chat_models import ChatOpenAI

# Step 1: Translate to target language
translate_prompt = PromptTemplate(
    input_variables=["text", "target_language"],
    template="Translate the following to {target_language}:\n{text}"
)
translate_chain = LLMChain(llm=llm, prompt=translate_prompt)

# Step 2: Check for cultural appropriateness
review_prompt = PromptTemplate(
    input_variables=["translated_text"],
    template="Review this translation for cultural appropriateness and suggest improvements:\n{translated_text}"
)
review_chain = LLMChain(llm=llm, prompt=review_prompt)

# Step 3: Finalize translation
finalize_prompt = PromptTemplate(
    input_variables=["translated_text", "feedback"],
    template="Apply the following feedback to finalize the translation:\nOriginal: {translated_text}\nFeedback: {feedback}"
)
finalize_chain = LLMChain(llm=llm, prompt=finalize_prompt)

# Compose the chain
full_chain = translate_prompt | translate_chain | review_chain | finalize_chain

# Execute
result = full_chain.invoke({
    "text": "Let's grab some coffee",
    "target_language": "Japanese"
})

Chains are easier to debug and maintain because the flow is explicit. When a chain fails, you can identify exactly which step caused the problem. Chains are also easier to test because you can verify each step independently.

When to Use Chains

  • Tasks with well-defined, linear workflows
  • Situations where you need full auditability of each step
  • High-volume, low-complexity transformations
  • When cost is a primary constraint
  • When deterministic behavior is required
Watch Out: Chains Can Still Be Non-Deterministic

Even within a deterministic chain, individual LLM calls can produce variable outputs. A "deterministic" chain only means the workflow steps are fixed; each step can still produce different outputs. Don't assume chain outputs are reproducible unless you also fix randomness (use temperature=0 or deterministic prompts).

Agents: Dynamic Decision-Making

Agents are necessary for complex tasks where the path from input to output cannot be predicted in advance. Agents excel at tasks that require planning, tool use, and dynamic decision-making. The agent observes its environment, decides what action to take, executes the action, and iterates until the task is complete.

Case Study: Research Agent
from langchain.Agents import AgentExecutor, create_openai_functions_agent
from langchain import hub

# Define tools the agent can use
tools = [
    Tool(name="search", func=search_api.run, description="Search for information"),
    Tool(name="url_open", func=browse.run, description="Open a URL and extract content"),
    Tool(name="calculator", func=calculate.run, description="Perform calculations"),
]

# Pull the prompt from LangChain hub
prompt = hub.pull("hwchase17/openai-functions-agent")

# Create the agent
agent = create_openai_functions_agent(llm, tools, prompt)

# Create the executor
agent_executor = AgentExecutor(
    agent=agent,
    tools=tools,
    verbose=True,
    max_iterations=10,
    max_execution_time=120
)

# Execute - agent decides what to do
result = agent_executor.invoke({
    "input": "What is the current market cap of Tesla? Compare it to Toyota's market cap and calculate the ratio."
})

# The agent will:
# 1. Recognize it needs current financial data
# 2. Use search tool to find Tesla market cap
# 3. Use search tool to find Toyota market cap
# 4. Use calculator to compute the ratio
# 5. Format the final answer

This makes Agents ideal for research tasks, complex problem-solving, and any scenario where the appropriate action depends on the results of previous actions.

When to Use Agents

  • Tasks where the steps cannot be predetermined
  • Research and information gathering tasks
  • Complex problem-solving requiring multiple tool uses
  • Situations where adapting to user input is essential
